# Validation Plan

WanderAware is currently in startup validation mode. The landing page and dashboard should support outreach learning before full IoT ingestion is live.

## Current Priorities

- Track outreach links with UTM parameters and non-PII lead identifiers.
- Measure landing page views, CTA clicks, form starts, waitlist submits, reservation attempts, and reservation submits.
- Keep the waitlist CTA lighter than reservation while preserving reservation as a higher-intent action.
- Use dashboard scaffolding to demo building, device, participant, tag, alert, team, billing, and admin flows.

## Product Boundaries

- WanderAware is a supplemental alerting system, not a replacement for direct staff supervision.
- Store operational status/risk fields, not diagnosis details, in v1 participant records.
- IoT reader ingestion is planned after the web app and validation funnel are stable.
